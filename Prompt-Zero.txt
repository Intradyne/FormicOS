SECTION 3: YOUR TESTING PROTOCOL
================================================================================

Before committing ANY code change, you MUST validate it against the existing test suite.

STEP 1 — Unit tests (always):
  python -m pytest tests/ -x --tb=short

STEP 2 — Targeted validation (when modifying a specific module):
  python -m pytest tests/test_<module>.py -xvs

STEP 3 — Version-specific regression tests:
  python -m pytest tests/test_v079_refactor.py -xvs   # Verify v0.7.9 refactoring intact
  python -m pytest tests/test_v078_harness.py -xvs     # Verify test flight harness intact

The `mock_llm` fixture in `tests/conftest.py` intercepts ALL outbound HTTP requests to:
  - https://api.openai.com/v1/chat/completions
  - https://api.anthropic.com/v1/messages

It returns deterministic JSON with zero token cost. You MUST use this fixture in any test that exercises LLM inference paths. NEVER call a real LLM endpoint during testing.

The `mock_config` fixture loads a minimal `formicos.yaml` from a temp directory. Use it to create FastAPI app instances for integration testing:
  from src.server import create_app
  app = create_app(mock_config)

The `async_client` fixture provides an `httpx.AsyncClient` wrapping the FastAPI app for HTTP endpoint testing without a running server.

YOU MUST NOT:
  - Add required fields to existing Pydantic models without defaults
  - Change the embedding model dimensions without a versioned Qdrant collection migration
  - Remove backward-compatible re-exports from src/server.py (tests depend on them)
  - Run live LLM inference in tests (use mock_llm)
  - Delete or overwrite .formicos/ session data without a migration script
  - Skip running pytest after code changes
  - Import from src.core/ or src.services/ in production code yet
    (they are re-export scaffolds; actual code lives in src/*.py flat files)
  - Call an `async def` method without `await` — STRICTURE-001.
    (e.g., `data = context_tree.to_dict()` when to_dict is async returns a coroutine, not a dict)
  - Use synchronous blocking I/O (json.dump, open, Path.rglob) inside async route handlers
    or asyncio.Lock blocks without asyncio.to_thread() wrapping — STRICTURE-002.
  - Type core class dependencies (ColonyManager, Orchestrator, AgentFactory constructors)
    as `typing.Any` — use Protocol classes from models.py instead — STRICTURE-003.

YOU MUST ALWAYS:
  - Run `python -m pytest tests/ -x` after every meaningful code change
  - Maintain 0 test failures (current baseline: 1231 passed, 2 skipped)
  - Use `is_test_flight=True` for any live colony debugging
  - Follow the Pydantic evolution rules in Data-Migration-Strategy.md
  - Bump VERSION in src/server.py and __version__ in src/__init__.py together
  - Verify async/sync nature of any method before calling it — inspect the `def` vs
    `async def` signature in the source file, especially for duck-typed/hasattr patterns
  - Wrap blocking I/O with `asyncio.to_thread()` in async contexts (see context.py:596
    for the canonical correct pattern)
