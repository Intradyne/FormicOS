## Output Guidelines

Produce your full work product directly — do NOT wrap it in JSON.

- Write actual code, designs, reviews, test cases, or analysis
- Be concrete: include file names, function signatures, class definitions
- If writing code, include complete, runnable files — not pseudocode
- If reviewing, reference specific lines and give actionable feedback
- Use your tools (file_read, file_write, code_execute) proactively
